## 📋 配置格式说明

每个卡片配置的格式：

```javascript
{ i: 索引号, t: "卡片标题", u: "页面路径" }
```

- **`i`**: 索引号（从0开始，依次递增）
- **`t`**: 卡片标题（显示在左侧边栏）
- **`u`**: 页面文件路径（相对于index.html）

**示例**：
```javascript
{ i: 15, t: "粒子系统", u: "pages/page16.html" }
```

## 🔢 索引号规则

**重要**：索引号必须从 **0** 开始，**连续递增**，不能跳跃！

### ✅ 正确示例
```javascript
{ i: 0, t: "卡片1", u: "pages/page1.html" },
{ i: 1, t: "卡片2", u: "pages/page2.html" },
{ i: 2, t: "卡片3", u: "pages/page3.html" }
```

### ❌ 错误示例
```javascript
{ i: 0, t: "卡片1", u: "pages/page1.html" },
{ i: 2, t: "卡片2", u: "pages/page2.html" },  // ❌ 跳过了 i: 1
{ i: 5, t: "卡片3", u: "pages/page3.html" }   // ❌ 跳过了 i: 3, 4
```

## 📐 轮盘布局自动适配

项目会根据卡片数量自动调整轮盘布局：

- **15张卡片**：每张相隔 24° (360° ÷ 15)
- **20张卡片**：每张相隔 18° (360° ÷ 20)
- **30张卡片**：每张相隔 12° (360° ÷ 30)

**完全自动，无需手动调整！**

## ⚠️ 注意事项

### 1. 文件路径必须正确
确保 `u` 字段中的路径对应的文件存在：
```javascript
{ i: 15, t: "新卡片", u: "pages/page16.html" }
//                        ↑ 确保这个文件存在
```

### 2. 索引号必须连续
从 0 开始，不能跳跃，否则可能导致切换异常。

### 3. 卡片数量建议
- **推荐**: 10-30 张卡片
- **最佳体验**: 15-20 张卡片
- **理论上限**: 无限制（但太多会影响性能和浏览体验）

### 4. 保存后记得刷新
修改 `index.html` 后，必须刷新浏览器（F5）才能看到效果。

## 💡 进阶：使用JSON配置（可选）

如果你希望更方便地管理卡片配置，可以考虑从外部JSON文件加载。

### 当前方式（硬编码）
```javascript
// index.html 中直接写
const slides = [
  { i: 0, t: "文字呼吸", u: "pages/page1.html" },
  // ...
];
```

### 优化方式（JSON配置）
```javascript
// index.html 中动态加载
fetch('data/slides.json')
  .then(res => res.json())
  .then(data => {
    const slides = data.slides;
    // ... 初始化轮盘
  });
```

**优势**：
- ✅ 修改配置不用碰HTML代码
- ✅ 更易于维护和版本控制
- ✅ 可以用工具批量生成配置

**目前项目中已有 `data/slides.json` 文件，但未使用。如需实现，请联系开发人员。**
